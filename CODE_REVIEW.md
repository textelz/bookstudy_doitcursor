# 프런트엔드 코드 리뷰 및 개선 사항

## 🔍 발견된 문제점 및 개선 사항

### 1. **중요한 버그**

#### 1.1 재고 데이터가 localStorage에 저장되지 않음
- **문제**: 재고 수량을 변경해도 페이지를 새로고침하면 초기값으로 돌아감
- **위치**: `AdminPage.jsx`
- **해결**: 재고 데이터도 localStorage에 저장/로드하도록 수정 필요

#### 1.2 Cart 컴포넌트에서 index를 key로 사용
- **문제**: React에서 배열의 index를 key로 사용하면 성능 문제와 버그 발생 가능
- **위치**: `Cart.jsx` line 47
- **해결**: 고유한 ID를 사용하거나 item의 고유 조합을 key로 사용

#### 1.3 주문 ID 충돌 가능성
- **문제**: `Date.now()`를 ID로 사용하면 동시에 여러 주문이 들어올 때 충돌 가능
- **위치**: `OrderPage.jsx` line 124
- **해결**: UUID 또는 더 안전한 ID 생성 방식 사용

### 2. **성능 이슈**

#### 2.1 불필요한 재계산
- **문제**: `calculateStats()`가 매 렌더링마다 실행됨
- **위치**: `AdminPage.jsx` line 22-31
- **해결**: `useMemo`로 최적화

#### 2.2 1초마다 localStorage 폴링
- **문제**: 관리자 화면이 활성화되지 않아도 계속 폴링함
- **위치**: `AdminPage.jsx` line 86-88
- **해결**: 커스텀 이벤트로 충분하므로 폴링 간격을 늘리거나 제거

### 3. **사용자 경험 개선**

#### 3.1 alert 사용
- **문제**: 브라우저 기본 alert는 사용자 경험이 좋지 않음
- **위치**: `OrderPage.jsx`, `Cart.jsx`
- **해결**: 토스트 메시지나 모달 컴포넌트로 대체

#### 3.2 이미지 로드 실패 처리 없음
- **문제**: 이미지가 없거나 로드 실패 시 에러 표시 없음
- **위치**: `ProductCard.jsx` line 36
- **해결**: 에러 핸들링 및 플레이스홀더 추가

#### 3.3 로딩 상태 표시 없음
- **문제**: 데이터 로드 중 상태 표시 없음
- **해결**: 로딩 스피너 또는 스켈레톤 UI 추가

### 4. **코드 품질**

#### 4.1 하드코딩된 데이터
- **문제**: 상품 데이터와 재고 데이터가 컴포넌트에 하드코딩됨
- **위치**: `OrderPage.jsx`, `AdminPage.jsx`
- **해결**: 별도 데이터 파일이나 API로 분리

#### 4.2 에러 처리 부족
- **문제**: localStorage 파싱 실패, JSON 파싱 실패 등에 대한 에러 처리 부족
- **해결**: try-catch 블록 추가 및 사용자 친화적 에러 메시지

#### 4.3 타입 안정성 없음
- **문제**: JavaScript만 사용하여 타입 체크 불가
- **해결**: TypeScript 도입 고려 (선택 사항)

### 5. **기능 개선 제안**

#### 5.1 재고 자동 차감
- 주문이 완료되면 해당 상품의 재고를 자동으로 차감하는 기능

#### 5.2 주문 취소 기능
- 관리자가 주문을 취소할 수 있는 기능

#### 5.3 주문 내역 상세 보기
- 주문을 클릭하면 상세 정보를 볼 수 있는 모달

#### 5.4 검색 및 필터링
- 주문 현황에서 상태별로 필터링하는 기능

#### 5.5 데이터 내보내기
- 주문 데이터를 CSV나 JSON으로 내보내는 기능
